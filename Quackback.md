# Quackback

Open-source alternative to Canny/UserVoice/Productboard, with built-in AI and — critically — an **MCP server** that lets agents in Claude/Cursor/Windsurf search, triage, and act on feedback directly. The most agent-native OSS project found for [[The Full Discovery Loop]].

## Project

- **Repo:** [github.com/QuackbackIO/quackback](https://github.com/QuackbackIO/quackback) — site: quackback.io
- **License / metadata:** `[Confirmed]` AGPL-3.0, TypeScript, **107★ / 27 forks** (GitHub API, 2026-05-28). Created 2025-12-02; actively maintained.
- **Indie status:** `[Inference]` Launched via Show HN and a DEV.to intro by author "mortondev" (~Feb 2026); appears to be a solo-founder effort. Solo status inferred from the single-author pattern, not a funding statement.
- **Model:** open-source self-host (Docker/Railway one-click); likely future hosted tier (`[Unverified]`).

## Loop fit

`[Confirmed]` Full feedback workflow (voting boards, roadmap, changelog) **plus** AI summaries, automatic duplicate detection, feedback extraction from external sources, and the MCP server for agent triage. Strong among OSS options — it makes feedback *queryable by coding agents*, which is exactly the integration [[The Full Discovery Loop]] calls for.
